Accessibility statement

{{ content_metadata( data={ "Published": "23 September 2020", "Last updated": "13 March 2024", "Next review due": "30 April 2025" } ) }} {# Last non-functional changes: 13 December 2024 #}

This accessibility statement applies to the www.notifications.service.gov.uk domain. It does not apply to the GOV.UK Notify API documentation subdomain.

This website is run by the GOV.UK Notify team at the Government Digital Service (GDS). It is designed to be used by as many people as possible. You should be able to:

  • change colours, contrast levels and fonts using browser or device settings
  • zoom in up to 400% without the text spilling off the screen
  • navigate most of the website using a keyboard or speech recognition software
  • listen to most of the website using a screen reader (including the most recent versions of JAWS, NVDA and VoiceOver)

We’ve also made the website text as simple as possible to understand.

AbilityNet has advice on making your device easier to use if you have a disability.

How accessible this website is

We know some parts of this website are not fully accessible:

  • the addition of new reply-to addresses is not announced correctly by screen readers
  • the Notify status page has several accessibility issues

We also know that the emails Notify sends are not fully accessible:

  • emails sent by Notify require scrolling when zoomed past 220% in Microsoft Outlook
  • the NVDA screen reader announces extra spaces in emails viewed in Microsoft Outlook

Feedback and contact information

If you need any part of this service in a different format like large print, audio recording or braille, contact the GOV.UK Notify team.

We’ll consider your request and get back to you by the end of the next working day.

Reporting accessibility problems with this website

We’re always looking to improve the accessibility of this website. If you find any problems not listed on this page or think we’re not meeting accessibility requirements, contact the GOV.UK Notify team.

Enforcement procedure

The Equality and Human Rights Commission (EHRC) is responsible for enforcing the Public Sector Bodies (Websites and Mobile Applications) (No. 2) Accessibility Regulations 2018 (the ‘accessibility regulations’). If you’re not happy with how we respond to your complaint, contact the Equality Advisory and Support Service (EASS).

Technical information about this website’s accessibility

GDS is committed to making its websites accessible, in accordance with the Public Sector Bodies (Websites and Mobile Applications) (No. 2) Accessibility Regulations 2018.

Compliance status

We have tested the Notify website against the Web Content Accessibility Guidelines (WCAG) 2.2 AA standard.

This website is partially compliant with the Web Content Accessibility Guidelines version 2.2 AA standard, due to a non-compliance listed below.

Non-accessible content

Non-compliance with the accessibility regulations

The following content on the Notify website is not compliant with the WCAG version 2.2 AA standard:

The notice telling users that they have successfully added a reply-to address is not being announced correctly by screen readers which fails success criterion 4.1.3: Status messages.

Emails

The following content in emails sent by Notify is not compliant with the WCAG version 2.2 AA standard:

Emails zoomed past 220% on the Microsoft Outlook application overflow the screen and so need scrolling to read which fails WCAG success criterion 1.4.10: Reflow.

The NVDA screen reader also announces extra spaces in emails when viewed in Microsoft Outlook. This does not fail WCAG but is known to cause confusion for users of NVDA.

Status page

The following content on the Notify status page is not compliant with the WCAG version 2.2 AA standard:

  • an incorrect heading hierarchy which fails success criterion 1.3.1: info and relationships
  • missing level 1 headings (users may not be able to accurately determine the structure of content on the page)
  • our status incident page is missing the main heading and the status home page has an incorrect heading hierarchy – this fails success criterion 1.3.1: info and relationships
  • our status page subscribe to updates forms, when submitted with incorrect data, do not inform screen reader users of the error message when it becomes available – this fails success criterion 4.1.3: status messages

How we tested this service

We revised the list of common tasks that GOV.UK Notify users need to complete and tested them against WCAG 2.2 in September 2024. We fixed the issues identified in September and tested them again in February 2025. We are now working on any new issues identified in the latest round of testing.

The parts of the website used to add email branding or letter branding were first tested in October 2023. Fixes to the button for copying text in the page to your clipboard and a new design of the component for choosing the day and time to send notifications were also tested as part of this.

The parts of the website used to send text messages, emails and letters were first tested in July 2020 and the resulting fixes tested in January 2021. The emails we send out, the web pages users see when they are sent a file by Notify and a sample letter PDF were last tested on 4 March 2019.

All tests were carried out by the Digital Accessibility Centre.

We originally decided which pages to test based on the most common tasks that GOV.UK Notify users need to complete. We also tested a sample of pages with common user interface components. Since then, we test any new features or substantial changes to the service. We also aim to review the common tasks and retest them every 2 years.
